# Grid Editor Redesign — Cell-Based DnD

**Дата:** 2026-03-17
**Статус:** Утверждено

## Проблема

Текущий конструктор вишлистов использует auto-fill сетку — блоки автоматически заполняют пустые ячейки, нет свободы размещения. Пользователь не может создать раскладку «ёлочкой», оставить пустое место, или точно контролировать, куда встанет блок.

## Решение

Ячеечная сетка 2 колонки с координатной моделью `(row, col)`. Каждый блок знает свою позицию в сетке. Пустые ячейки допустимы и видимы. DnD — перетаскивание в конкретную ячейку.

## Модель данных

### Block type — изменения

**Убираем:**
- `position: number`
- `mobilePosition?: number`
- `columnStart?: 1 | 2`
- `rowSpan?: 1 | 2 | 3`

**Добавляем:**
- `row: number` — 0-based, номер строки
- `col: 0 | 1` — 0 = левая, 1 = правая (игнорируется при colSpan: 2)

**Оставляем:**
- `type: BlockType`
- `colSpan: 1 | 2` — 1 = одна ячейка, 2 = на всю ширину
- `data: Record<string, unknown>`

### API контракт

`PUT /wishlists/:id/blocks` — тело по-прежнему `Block[]`. Бэкенд сохраняет `row`/`col`/`colSpan` как есть. Порядок в массиве больше не определяет позицию — координаты явные.

### Миграция

Фронтенд при загрузке:
1. Если у блока есть `row`/`col` → новый формат, используем как есть
2. Если нет → прогоняем через текущий `computeGridPositions()`, вычисляем row/col из результата
3. При первом сохранении пишется новый формат

Бэкенд: сохраняет дополнительные поля. Старые поля можно игнорировать.

## Сетка

### Рендеринг

- CSS Grid: `grid-template-columns: 1fr 1fr`, gap: 12px
- Количество строк: `maxRow + 2` (всегда +1 пустая строка внизу)
- Блоки позиционируются: `gridRow: row + 1`, `gridColumn: col + 1 / span colSpan`
- Линии сетки видны всегда (пунктирные рамки на пустых ячейках)

### Пустые ячейки

- Каждая пустая ячейка — `useDroppable` с id `cell-{row}-{col}`
- Визуально: пунктирная рамка + лёгкий полупрозрачный фон
- При наведении перетаскиваемого блока → ярче подсвечивается

## Drag & Drop

### Библиотека

- Оставляем `@dnd-kit/core`
- Убираем `@dnd-kit/sortable` — не нужна сортировка
- Блоки = `useDraggable`, ячейки (пустые и занятые) = `useDroppable`
- Collision detection: `closestCenter`

### Сценарии

**1. Дроп в пустую ячейку:**
- Блок перемещается: `block.row = targetRow`, `block.col = targetCol`
- Старая ячейка становится пустой

**2. Дроп в занятую ячейку (push):**
- Блок из целевой ячейки и все ниже (`row >= targetRow`) сдвигаются `row += 1`
- Перетаскиваемый блок встаёт на освобождённое место
- Старая ячейка — пустая

**3. Дроп wide-блока (colSpan: 2):**
- Wide-блок бросается только с col: 0
- Если в целевой строке есть блоки — они сдвигаются вниз (`row += 1`)
- Целевая строка занята целиком

**4. Добавление из палитры:**
- Новый блок добавляется в первую свободную ячейку (scan row→col)
- Или drag из палитры прямо в целевую ячейку

### Подсветка при перетаскивании

- Зелёная рамка + фон = текущая целевая ячейка
- Синий пунктир = все доступные drop-зоны
- Жёлтый = блоки, которые будут сдвинуты (при дропе в занятую ячейку)

## Мобильная версия

Убираем переключатель Десктоп/Мобила. Мобильная версия (публичная страница) автоматически сворачивает сетку:

```
blocks.sort((a, b) => a.row - b.row || a.col - b.col)
```

Одна колонка сверху вниз, пустые ячейки пропускаются. Никакого отдельного `mobilePosition`.

## Визуальные подсказки

- **Линии сетки всегда видны** — пунктирные рамки на всех ячейках
- **Подсветка drop-зон** при перетаскивании (описано выше)
- **Онбординг-тултипы** при первом редактировании:
  - «Перетащи блок в любую ячейку»
  - «Пустые места — это нормально»
  - «Растяни блок на всю ширину через тулбар»

## Что меняется в файлах

- `shared/types.ts` — Block type
- `app/wishlist/components/constructor/block-canvas.tsx` — полная переделка: координатная сетка, droppable-ячейки, новая DnD логика
- `app/wishlist/components/constructor/block-item.tsx` — useDraggable вместо useSortable
- `app/wishlist/components/constructor/block-toolbar.tsx` — убрать rowSpan пилюли, оставить colSpan toggle
- `app/s/[shortId]/components/presents-grid.tsx` — мобильная авто-свёртка (если используется для блоков)
- Удалить `@dnd-kit/sortable` из зависимостей

## Что НЕ меняется

- Палитра блоков (block-palette.tsx) — добавление как раньше
- Модальное редактирование блоков (block-editor-modal)
- API хуки (api/wishlist/) — тот же endpoint
- Бэкенд — только принимает новые поля, ничего не ломаем
